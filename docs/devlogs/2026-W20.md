- `TP Flash` 的 pressure / temperature phase-region 分类不应各自演化一套容差常量；窗口调和只能复用 `rf-types` 的正式 tolerance 语义，并且只处理边界容差带内的尺度差异，不把真实压力 / 温度分类冲突吞掉。

- Stream Inspector 中 `Draft` 只表示未提交草稿；已进入 document 但组成和不为 1 的流股应以 `Unnormalized` 暴露，并继续由显式 normalize 或运行前阻断处理，不在 add/remove 或 run command 中自动差值补偿。
- command surface 对 `SolveSnapshot` result navigation 的暴露应是 presentation 层命令注册：命令 id 继续复用 `inspector.focus_stream:*` / `inspector.focus_unit:*`，执行仍走 host `dispatch_ui_command`，不为 palette、menu、command list 或 runtime 按钮各自解释一套目标语义。
- runtime 内部的小型 command action button 应继续只是 `StudioGuiWindowCommandActionModel` 的渲染与点击派发器；点击后的目标解释仍归 host `dispatch_ui_command` / inspector target driver，不在 runtime renderer 内复制 `inspector.focus_*` 解析。
- MVP 第一阶段当前应视为进入 α 验收准备：M1-M5 已有可验证基线，后续优先验收矩阵、仓库级验证和用户视角 smoke；除非验收路径暴露 blocker，不再主动扩 shell 消费面细节测试。

- 窗口获得焦点不应在 MVP 本地 smoke 中自动触发 entitlement/timer 检查；这类生命周期检查应保留在显式命令、真实 timer callback 或后续受控平台集成路径里，避免启动阶段抢跑运行时 dispatch。
- 纯 UI 默认布局不应在启动阶段通过完整 runtime dispatch 追补；shell-only 的默认呈现偏好应落在 host-local transient layout 层，避免初始化烟测抢跑 bootstrap trigger，也避免改动底层 layout 算法默认语义。
