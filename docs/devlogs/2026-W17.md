## 关键决策

- 本轮不把 `UnitOperationSmokeHostDriver` 整体上移为正式库 API
- 正式上移的边界是“action execution request planning”，不是“完整宿主生命周期编排”
- request planner 只消费宿主显式提供的 parameter values 与 port objects，不替宿主选择 flowsheet JSON、package id、连接对象命名或 lifecycle 调用时机
- smoke host / 未来 PME host / 其他宿主入口后续应优先复用 action plan、request planner、dispatcher 与 session snapshot，而不是各自重写 action 分类和输入缺失判断
- smoke host / 未来 PME host / 其他宿主入口后续应优先复用 action plan、request planner、host view reader、orchestrator、validation/calculation runner、host round outcome、supplemental mutation phase、follow-up 与 session snapshot，而不是各自重写 action 分类、输入缺失判断、optional mutation 注入、phase gating 和执行后下一步建议
- 未来 PME host / 其他宿主入口的最小接线应优先以 `SampleHost` 演示的正式消费路径为蓝本，而不是以 smoke driver 的验证 DSL 为蓝本
- 当前 PME-like 薄宿主入口已经证明 `SampleHost` 路径可以从“console 演示”前推为“薄宿主 session 形状”；host round 兜底子任务到这里应继续视为结束，不需要继续深挖
- 当前注册前置口径已先冻结身份和 dry-run 计划；下一步的必须补口不再是 CLSID/ProgID 选择，而是执行型 COM 注册工具的安全边界、安装/反安装路径与目标 PME 人工验证说明
- dry-run registry plan 当前用于先审查写入范围，后续若实现执行型注册，必须显式加入权限检查、路径存在性检查、备份/回滚策略和用户确认，不应直接从 plan 变成静默写注册表
- preflight 仍只允许读本机状态，不允许写 registry；真正写入必须另加显式执行门控，并单独验证回滚路径
- 目标 PME 人工验证说明已经足以结束当前“下一步如何进入真实 PME 前置准备”的文档缺口；后续必须补的是执行型注册实现和真实 PME 记录，不是继续扩 host round 兜底
- 当前子任务可正式从“host round 兜底 / 外部 host 消费路径证明”转为“M5 注册脚本固化与目标 PME 人工验证准备”；继续深挖 smoke driver 或 host round fallback 的收益已经低于推进注册入口、安装说明与 PME 记录

## 风险与未完成项

- 当前 request planner 仍只覆盖 MVP action kinds，不承诺完整 CAPE-OPEN PME 生命周期编排

- smoke driver 仍是验证型 helper，后续如果继续上移，需要先拆清真实宿主 API 与 smoke 场景 DSL 的边界
- `SampleHost` 当前虽已具备 PME-like 薄宿主 session，但仍不代表已经进入 COM 注册、PME 自动化互调或完整宿主管理框架
- `Registration` 当前虽已具备 execute 与仓库脚本入口，但验证范围仍只覆盖自有 `current-user register/unregister`；`local-machine`、PME 互调与第三方模型加载仍未进入当前阶段
- 当前已把安装/反安装说明与验证记录模板冻结到正式文档，但真实目标 PME 记录、`local-machine` 策略判断与 PME 互调本身仍未完成
- `DWSIM` 目标宿主虽已选定且注册现场已准备好，但 discovery / activation / validate / calculate 仍待人工在已启动的 GUI 中继续完成，当前新增记录先标记为 `Blocked`
- 当前 `UnitOp.Mvp` 已具备真实 `TLB / TypeLib` 产物、注册执行路径、assembly-level TLB identity 与默认 COM interface 口径；`Windows PowerShell 5` 默认 `ICapeUtilities` late-bound 探测已通过，但真实 `DWSIM / COFE` discovery / activation / validate / calculate 仍未复验
