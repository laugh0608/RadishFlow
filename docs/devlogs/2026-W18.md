- 这些能力仍属于最小 Runtime / Canvas presentation、正式 command / driver 边界和 shell 临时状态，不代表已经进入完整结果审阅、结果表格导出、跨会话结果历史、完整画布编辑器、端口连线编辑、坐标持久化或 UI 视觉精修阶段；后续若继续推进画布，应先补正式 presentation / command / state 边界，再扩真实 UI。

## 风险与未完成项

- `DWSIM / COFE` 验证仍是人工路径，不代表已经具备 PME 自动化互调测试。
- `IPersistStreamInit`、`IPersistStorage` 与 `IOleObject` 当前仍是 PME/OLE 探测所需的最小兼容面，不代表已经实现 PME 工程文件级持久化或完整 OLE embedding。
- 若未来 PME 要求长期读取/写入 material object，需要重新设计短生命周期边界，不能把 live PME material object 持久化进产品状态。
- Studio Runtime 面板当前只是最小可见结果面；虽然已补出结构化流股结果 presentation、最小 Result Inspector 与当前快照内流股对比，后续若要做更完整的结果审阅、结果表格导出或图形化流股检查，仍应先补更完整的 presentation / command 边界，再扩真实 UI。
- 当前布局修正只解决右侧 Runtime 可读性，不代表 Studio 已进入完整视觉设计阶段；后续若要继续重排主界面，应先明确面板信息架构和用户工作流。
- 项目打开入口当前支持输入路径、Windows 原生选择器和最近项目打开已有项目；最近项目已经具备独立 preferences 持久化，但尚未接入完整应用偏好系统。
- 保存 / 另存为已有最小闭环，项目文件写入也已具备 staged replace、Save As 覆盖确认和失败恢复状态保护；但原生文件选择器当前仍只覆盖 Windows 路径，更细的跨平台文件工作流、失败恢复文案和平台级原子替换能力仍应继续沿 `DocumentLifecycle` 与 `rf-store` 边界推进。
- Stream Inspector 字段现在已有草稿更新、单字段提交、多字段批量提交、基础 undo/redo、保存态语义与快捷键焦点归属；跨会话历史持久化仍未完成，后续继续沿 `CommandHistory` 与项目持久化边界推进，避免把这些语义塞回 shell 面板。
- 画布目前只补出前置 pending edit、取消命令、`egui` 单类型发起/点击提交入口、最小单元创建提交，以及已有单元/物流线/端口/诊断的只读扫读层；尚未实现拖拽放置、多单元 palette、端口连接编辑、自动布线或坐标布局持久化。后续应继续沿 `CanvasEditIntent -> presentation -> command -> DocumentCommand / layout state` 的顺序补齐，不直接在 `egui` shell 中写项目文档。
