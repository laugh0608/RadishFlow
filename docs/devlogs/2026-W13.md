## 关键决策

- Rust Core 与 `.NET 10` CAPE-OPEN 适配层严格解耦
- 第一阶段只导出自有 Unit Operation PMC，不支持第三方模型加载
- 当前优先级压在 `M1/M2`，不提前推进 `.NET` 复杂实现和 UI 扩展
- 开发日志采用按周分文件维护，便于阶段回顾和任务切片
- 协作规范细化为可执行规则，并与当前仓库阶段边界保持一致
- 在恢复主线功能推进前，先完成仓库治理、CI 和 App 架构这类地基工作
- dev 作为常态开发分支，master 只接受 PR，不对 dev 增加当前阶段不必要的保护
- App 架构当前采用“HYSYS 式交互 + 显式状态分层 + 独立结果快照”的折中路线
- `FlowsheetDocument` 正式收紧为可序列化真相源；文件路径、保存修订号和草稿态不再混入文档元数据
- `CommandHistory` 正式只记录语义化文档命令；运行控制和文档生命周期动作不进入撤回栈
- `SolveSessionState` 正式绑定当前观察的文档修订号，`SolvePendingReason` 只表达待办原因，不复用为错误容器
- `rf-ui` 本轮先只复用 `rf-model::Flowsheet` 和 `rf-types` 基础 ID；`DocumentId` / `SolveSnapshotId` 等 App 本地类型暂留在 UI 层，不提前上推到 Core
- RadishFlow 当前正式采用“本地求解 + 外部控制面”的混合方案：OIDC 登录、授权、派生资产分发走远端，主求解热路径继续保留在本地
- 桌面端统一视为 `public client`，后续采用系统浏览器 + PKCE + OS 安全存储，而不是内嵌长期密钥或浏览器本地存储口径
- 项目文件真相源与授权缓存真相源当前正式拆开：`rf-store` 不再以单一根对象混装项目文档、授权缓存和安全凭据
- `StoredProjectFile.kind = radishflow.project-file` 与 `StoredAuthCacheIndex.kind = radishflow.auth-cache-index` 当前正式作为两类 JSON 根对象标识
- 授权缓存索引当前正式只保存相对缓存路径与安全凭据引用；缓存根目录、项目文件路径和 token 明文继续保持分离
- `rf-thermo` 当前正式区分“求解热路径 provider”和“受控派生物性包 provider”，避免把联网授权逻辑压进求解接口
- 控制面 scope 当前正式收敛为细粒度资源命名：`radishflow.control.read`、`radishflow.assets.read`、`radishflow.assets.lease`、`radishflow.audit.write`
- `rf-store` 当前正式把时间戳 JSON 表达固定为 RFC3339 字符串，把缓存相对路径 JSON 表达固定为 `/` 分隔字符串，避免平台差异污染项目文件和缓存索引
- PR 检查当前正式收口为“仅对指向 `master` 的 Pull Request 自动触发”；`dev` 继续作为默认集成分支，但不因直接 push 自动跑检查
- RadishFlow 当前正式采用 `vYY.M.RELEASE` + `-dev / -test / -release` 的项目发布 tag 口径，并与规范 tag push 的 CI 触发条件保持一致
- 控制面接口当前正式开始从“文档草案”下沉为 `rf-ui` 运行时 DTO，对象粒度先冻结为 manifest list、lease grant、offline refresh 和 audit event
- 授权缓存桥接当前正式收口到 `apps/radishflow-studio`，`rf-ui` 与 `rf-store` 继续避免直接互相依赖
- 本地缓存包记录的 `expiresAt` 当前正式跟随授权快照/离线租约有效期，而不是短时下载 URL 的 `lease grant expiresAt`
- `rf-ui` 当前正式把 `EntitlementSnapshot.schemaVersion`、`PropertyPackageManifest.schemaVersion` 和 `leaseRequired` 补进运行时 DTO，避免正式契约与代码字段继续漂移
- 本地 `manifest.json` 当前正式固定为 `radishflow.property-package-manifest`，本地 `payload.rfpkg` 当前正式固定为 `radishflow.property-package-payload`，两者都采用带 `schemaVersion` 的 JSON DTO 进入第一版持久化
- `PropertyPackageProvider` 当前正式增加基于缓存根目录 + `StoredAuthCacheIndex` 的本地实现，并在构造阶段前置校验索引、manifest 和 payload 一致性
- 本地 provider 当前正式只暴露未过期且存在本地 payload 的包；`RemoteEvaluationService` 记录不进入本地可装载包列表
- 下载完成后的本地缓存写入当前正式收口到 `apps/radishflow-studio`：先写 payload，再写 manifest，最后写 auth cache index，避免未写完索引前就把半成品暴露成可发现状态
- 当前仓库已提供首个真实物性包样例 `binary-hydrocarbon-lite-v1`，后续新增包应优先复用这套 DTO 与目录口径
- 仓库许可采用 source-available 口径，并要求代码元数据与 `LICENSE` 文件保持一致
- 协作默认策略进一步收紧为“优先根治，不做无止境的治标兜底”
